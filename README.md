

## 1. Overview

It is a multi-RP RPKI measurement and analysis system for studying cross-operator view divergence.

## 2. Architecture

The system is structured into three layers:

### L1: Repository View
- RRDP notification layer


### L2: Object View
- Manifest
- ROA / CRL / certificate objects
### L2-b: Effective Validator Input
- Objects actually consumed by Routinator validator
- Cache-aware and cycle-aware input set

### L3: Output View
- Final VRP set generated by validator
- Cross-probe VRP divergence analysis
## 3. Core Components

- probe/ : 3-node distributed collectors (cd / bj / sg)
- collector/ : central aggregation (CD2)
- rp_adapters/ : RPKI validator and control-plane interface
- scripts/ : M17/M18/M19/M23B-F pipelines
- s3lib/ : shared utilities
- config/ : system configuration

---


## 4. Data Policy

- data/ is NOT tracked in GitHub
- logs/ are runtime-only
- outputs/ are derived artifacts
- experiments_archive/ contains historical debug runs

Only code + schema + tests + documentation are versioned.

---

## 5. Reproducibility

Each experiment run is identified by:

- window_id
- probe_id
- validator_cycle_id
- run_id

Ensures reproducible cross-RP comparison.
